Interactive Segmentation Tool for Symbios BoneSurfacer
CTI Segmentation project: Interactive editing

The project is written in C++ and takes advantage of Qt, VTK and OpenCV.

Compiling the project

Requirements

  • Qt Kit 5.5.0 or newer
  • CMake 3.10 or newer
  • built and installed copy of VTK (version 8)
  • built and installed copy of OpenCV (version 4)

Development environment

Using Qt Creator is recommended, but not mandatory. You can build the project from command line as any other CMake based project.

Compilation process (using Qt Creator)

The CMakelists.txt file in the project root serves as makefile (requires CMake).

  1. Checkout the repository.
    In the project directory CMakeLists.txt file should be present among others: it is the MAKEFILE for cmake describing the dependencies and build properties, it can be opened as project from the Qt Creator IDE.
  2. Open a project in Qt Creator's File menu, then select the CMakeLists.txt file. If all dependecies are found, CMake will take care of all the project configuration.
  3. Make sure you're pointing to the correct dynamic libraries for the version you're building the application. VTK contains Qt code: don't use VTK debug libraries when building the release or Qt will complain as folllows: QWidget: Must construct a QApplication before a QWidget
  4. Hit Build in the Qt Creator GUI and wait.
  5. Make sure that VTK and OpenCV dynamic libraries are system-visible (in the path) or copy them inside the built application folder, where the executable is. If some dynamic libraries are missing, running the application will result in an unknown error.
  6. Copy the Icons folder in the built application folder.
  7. Finally, Run the application.

    Deploying the application

Follow the compilation process carefully: make sure you copy the icons folder and that VTK and OpenCV dynamic link libraries are visible. In addition to VTK and OpenCV frameworks, the required Qt dynamic libraries are:

  • Qt5Core
  • Qt5GUI
  • Qt5Widgets

OpenCV is used to detect and remove the calibration phantoms. Only a few libraries are needed from OpenCV:

  • opencv_imgproc
  • opencv_core

VTK is the main tool used in the application, it is not possible to determine a subset of dynamic libraries required by the application. It is recommended to build it (using the same compiler and settings as for the application) and install it as a system package (f.ex using make install or mingw32-make install after building it).

Other dynamic libraries are required depending on the compiler used to build the application. If Mingw-w64 was used to build a Windows application, the required dynamic link libraries are:

  • gcc_s_seh-1
  • gomp-1
  • stdc++-6
  • winpthread-1
